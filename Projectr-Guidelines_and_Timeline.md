The goal of this project is to build something that you can showcase. A system, application, or demo that demonstrates a clear and compelling use of computer vision methods. Project format

Work in groups of 2–3 students (register your team on iCorsi).

Present your project at the end of the semester (format will follow).

The project will account for 20% of your final grade.

Deliverables

Code release (e.g., GitHub repository or shared files).

Live demo during the project presentation.

Poster and/or presentation (details follow closer towards the end of the semester).

Compute Resources

If you base your work on pre-trained networks, you might require less extensive amounts of computational resources. Suggested foundation models (optional but recommended)
You are encouraged to build on modern vision models. Below are some options with brief descriptions:

SAM: Segment objects in images or videos (interactively).

SAM 1 (images): <https://ai.meta.com/blog/segment-anything-foundation-model-image-segmentation/> | <https://arxiv.org/abs/2304.02643>

SAM 2 (images & videos) <https://ai.meta.com/research/sam2/> | <https://arxiv.org/abs/2408.00714>

SAM 3 (prompt-based) <https://ai.meta.com/research/sam3/> | <https://arxiv.org/abs/2511.16719>

TRELLIS/SAM-3D: Generate or extract 3D objects from images.

TRELLIS: <https://microsoft.github.io/TRELLIS/>

SAM-3D: <https://ai.meta.com/research/sam3d/> | <https://ai.meta.com/research/sam3d/>

DINO: Self-supervised visual feature learning, useful for clustering and representation. E.g., a strong image feature extractor to build anything on top.

<https://ai.meta.com/research/dinov3/> | <https://arxiv.org/abs/2508.10104>

CLIP: Connects images and text; embeds images and text into the same latent space so you can find image-text correspondences.

<https://openai.com/index/clip/> | <https://arxiv.org/abs/2103.00020>

SuperDec: Extract ultra-compact geometry proxies for complex 3D objects. Excellent for robotics and edge device deployment.

<https://super-dec.github.io/>

SceneScript: Extract a simplified text-based scene representation (floor plan layout and objects' bounding boxes).

<https://www.projectaria.com/scenescript/> | <https://arxiv.org/abs/2403.13064>

OpenMask3D: Open-vocabulary 3D instance segmentation (Point cloud to object instances with text-aligned, CLIP features).

<https://openmask3d.github.io/>

JEPA (Joint Embedding Predictive Architecture): Learns predictive representations without explicit supervision.

V-JEPA 2 (Video understanding for visual Q&A, action recognition/anticipation, ...): <https://ai.meta.com/research/vjepa/> | <https://arxiv.org/abs/2506.09985>

Locate 3D (Locate 3D objects in 3D scenes): <https://locate3d.atmeta.com/> | <https://arxiv.org/abs/2504.14151>

These models provide strong building blocks for ambitious and interesting projects, but you are free to use any other methods as well.
